## 核心讲解

### 1. 学习要点

条件语句用于根据布尔表达式选择执行路径。**算法中的分支，本质就是把问题分情况处理**。

### 2. 学习要点

**循环用于重复执行相同逻辑**。for 更适合遍历已有序列，while 更适合条件驱动的重复过程。

### 3. 学习要点

函数把一段逻辑封装成可复用单元。**清楚输入、输出和副作用，是写好算法函数的第一步**。

### 4. 学习要点

**异常用于表达不正常情况，例如空列表求最大值**。对非法输入主动报错，比让程序产生隐蔽错误更好。

### 任务 1：实现 count_positive(data)

要求：先自己写，再运行测试；运行后用一句话说明输入、输出和复杂度。

```python
def count_positive(data):
    count = 0
    for num in data:
        if num > 0:
            count += 1
    return count
```

```text
输入：data，一个数字列表
输出：列表中大于 0 的元素个数
时间复杂度：O(n)，因为需要遍历整个列表
空间复杂度：O(1)，因为只使用了 count 一个额外变量
```

### 任务 3：实现 safe_divide(a, b)

要求：先自己写，再运行测试；运行后用一句话说明输入、输出和复杂度。

```python
def safe_divide(a, b):
    if b == 0:
        return None
    return a / b
```

```text
输入：a 和 b，两个数字
输出：如果 b 不为 0，返回 a / b；如果 b 为 0，返回 None
时间复杂度：O(1)，因为只做一次判断和一次除法
空间复杂度：O(1)，因为没有使用随输入规模增长的额外空间
```

### 任务 4：给 find_max 增加空列表异常

要求：先自己写，再运行测试；运行后用一句话说明输入、输出和复杂度。

```python
def find_max(data):
    if not data:
        raise ValueError("empty sequence")
    max_value = data[0]
    for num in data:
        if num > max_value:
            max_value = num
    return max_value
```

```text
输入：data，一个非空数字列表
输出：列表中的最大值；如果列表为空，抛出 ValueError
时间复杂度：O(n)，因为需要遍历整个列表
空间复杂度：O(1)，因为只使用了 max_value 一个额外变量
```

## 边界测试是什么

**边界测试**，也叫 **边界值测试**，是一种软件测试方法。

它的核心思想是：

> 程序最容易在“临界点”出错，所以测试时要重点测边界附近的数据。

------

### 举例理解

假设一个输入年龄的功能要求：

```text
年龄必须在 18 到 60 岁之间
```

有效范围是：

```text
18 <= age <= 60
```

那么边界就是：

```text
18 和 60
```

边界测试会重点测试这些值：

| 测试值 | 含义           | 预期结果 |
| ------ | -------------- | -------- |
| 17     | 小于最小边界   | 不通过   |
| 18     | 最小边界       | 通过     |
| 19     | 刚进入有效范围 | 通过     |
| 59     | 接近最大边界   | 通过     |
| 60     | 最大边界       | 通过     |
| 61     | 超过最大边界   | 不通过   |

重点不是随便测几个值，而是测：

```text
边界值
边界值 - 1
边界值 + 1
```

------

### 为什么要做边界测试

很多程序错误都出现在边界条件上，例如：

```python
if age > 18 and age < 60:
    print("通过")
```

这段代码的问题是：

```text
18 和 60 都不能通过
```

但需求可能是：

```text
18 到 60 都可以通过
```

正确写法应该是：

```python
if age >= 18 and age <= 60:
    print("通过")
```

也可以简写为：

```python
if 18 <= age <= 60:
    print("通过")
```

边界测试就能发现这种错误。

------

### 常见边界测试场景

#### 1. 数值范围

例如：

```text
分数范围：0 到 100
```

测试：

```text
-1, 0, 1, 99, 100, 101
```

------

#### 2. 字符长度

例如用户名长度要求：

```text
用户名长度 6 到 20 位
```

测试：

```text
5 位
6 位
7 位
19 位
20 位
21 位
```

------

#### 3. 列表长度

例如购物车最多添加 99 件商品：

```text
0 件
1 件
98 件
99 件
100 件
```

------

#### 4. 日期边界

例如活动时间：

```text
2026-01-01 到 2026-01-31
```

测试：

```text
2025-12-31
2026-01-01
2026-01-02
2026-01-30
2026-01-31
2026-02-01
```

------


### 简单总结

**边界测试就是专门测试输入范围边缘的数据。**

常用口诀：

```text
最小值 - 1
最小值
最小值 + 1
最大值 - 1
最大值
最大值 + 1
```

它主要用来发现：

```text
> 和 >= 写错
< 和 <= 写错
数组越界
长度限制错误
日期范围错误
数量上限错误
```
