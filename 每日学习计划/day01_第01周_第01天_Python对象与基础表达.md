### 1.1.1 Python 解释器的作用

Python 是一种解释型语言。你写好的代码需要交给 **Python 解释器** 执行。


这表示让 Python 解释器运行 `demo.py` 文件。

### 1.2 Python 对象

Python 是面向对象语言。简单说，**Python 中几乎所有数据都是对象**。

例如：

```python
x = 10
name = "Alice"
data = [1, 2, 3]
```

这里的 `10`、`"Alice"`、`[1, 2, 3]` 都可以看成对象。变量名 `x`、`name`、`data` 是用来引用这些对象的名字。

这一节是后面学习链表、树、图的基础，因为这些结构本质上都要处理对象之间的引用关系。

> 个人理解：可以把变量想成一个箱子，你可以往这个箱子里面放任何东西，每个箱子都有自己的名字，在引用变量时，就相当于把这个箱子搬过来，你就能得到箱子里面你需要的东西。装了什么东西箱子会变成对应的样子，比如装了数字的变成“数字箱”，装了字符串的变成“字符箱”等等，所以要合并箱子里面的东西就要先转换箱子的类型，保持一致才行。

### 1.2.1 标识符、对象和赋值语句

赋值语句是 Python 中最基本、也最重要的语句之一。

```python
temperature = 98.6
```

这行代码的意思不是“把 98.6 放进 temperature 盒子里”，更准确地说是：**让标识符 `temperature` 引用一个值为 `98.6` 的浮点数对象**。

这里有三个概念：

- **标识符**：变量名，比如 `temperature`
- **对象**：实际的数据，比如浮点数对象 `98.6`
- **赋值**：让变量名引用某个对象

Python 的标识符有几个规则：

- **大小写敏感**，`age` 和 `Age` 是两个不同名字
- 可以由字母、数字、下划线等组成
- 不能以数字开头
- 不能使用 Python 保留字，比如 `if`、`for`、`class`

Python 是动态类型语言。也就是说，**变量名本身不固定类型，对象才有类型**。

例如：

```python
x = 10
x = "hello"
```

这在 Python 中是合法的。第一次 `x` 引用整数对象，第二次 `x` 改为引用字符串对象。

还有一个非常重要的概念：**别名**。

```python
a = [1, 2, 3]
b = a
```

这时 `a` 和 `b` 引用的是同一个列表对象。它们是同一个对象的两个名字。

所以：

```python
b.append(4)
print(a)
```

输出会是：

```python
[1, 2, 3, 4]
```

因为 `b.append(4)` 修改的是那个共同的列表对象，`a` 当然也能看到修改后的结果。

但是如果重新赋值：

```python
a = [1, 2, 3]
b = a
a = [9, 9, 9]
```

这不是修改原列表，而是让 `a` 改去引用一个新列表。`b` 仍然引用原来的 `[1, 2, 3]`。

这一点后面极其重要：**修改对象** 和 **让变量名改指向另一个对象** 是两件不同的事。

### 1.2.2 创建和使用对象

创建某个类的新对象，叫作 **实例化**。

例如：

```python
data = list()
```

这里调用 `list` 类的构造函数，创建了一个新的空列表对象。

很多内置类型也可以用字面量创建：

```python
x = 10          # int 对象
y = 3.14        # float 对象
name = "Bob"    # str 对象
data = [1, 2]   # list 对象
```

对象通常有自己的方法。方法可以理解成“属于某个对象的函数”。

例如：

```python
data = [3, 1, 2]
data.sort()
```

`sort()` 是列表对象的方法。它会修改列表本身，让列表变成有序。

有些方法只读取对象信息，不修改对象，这类方法可以理解成 **访问器**。

有些方法会改变对象状态，比如 `list.append()`、`list.sort()`，这类方法可以理解成 **更新方法**。

重点区分：

- **访问器方法**：读取信息，不改变对象
- **更新方法**：改变对象内部状态

例如：

```python
data = [3, 1, 2]
print(len(data))  # 读取长度，不改变 data
data.append(4)    # 修改 data
```

### 1.2.3 Python 的内置类

Python 提供了许多常用内置类。今天先重点认识这些：

```python
bool 布尔
int 整型
float 浮点型
list 列表
tuple 元组
str 字符串
set 集合
frozenset 不可变集合
dict 字典
```

可以从“是否可变”来理解它们。

**不可变类型**：对象创建后，值本身不能被修改。

- `bool`
- `int`
- `float`
- `tuple`
- `str`
- `frozenset`

**可变类型**：对象创建后，内部内容可以被修改。

- `list`
- `set`
- `dict`

这个区别非常重要。

例如整数是不可变的：

```python
x = 10
x = x + 1
```

这里不是把原来的整数对象 `10` 改成 `11`，而是创建或取得一个新的整数对象 `11`，然后让 `x` 引用它。

列表是可变的：

```python
data = [1, 2, 3]
data.append(4)
```

这里是直接修改原来的列表对象。

几个核心类型的用途：

- **bool**：表示 `True` 或 `False`
- **int**：表示整数，可以很大
- **float**：表示浮点数
- **list**：表示有顺序、可修改的一组对象
- **tuple**：表示有顺序、不可修改的一组对象
- **str**：表示字符串，不可修改
- **set**：表示无重复元素的集合，适合快速判断元素是否存在
- **dict**：表示键值映射，适合根据 key 查找 value

从数据结构角度看：

- **list 是后面数组序列的入口**
- **set 和 dict 背后与哈希表有关**
- **tuple 和 str 是不可变序列**
- **dict 是映射结构的直接例子**

### 1.3 表达式、运算符和优先级

表达式是能够计算出一个值的代码。

例如：

```python
3 + 5
10 > 4
len([1, 2, 3])
```

运算符的含义会受到操作数类型影响。

例如：

```python
3 + 5
```

这里 `+` 表示数字加法。

```python
"hello" + "world"
```

这里 `+` 表示字符串连接。

所以要记住：**同一个运算符，在不同类型对象上可能有不同含义**。

### 逻辑运算符

Python 有三个主要逻辑运算符：

```python
not
and
or
```

含义：

- `not`：逻辑非
- `and`：逻辑与
- `or`：逻辑或

例如：

```python
age = 20
print(age >= 18 and age <= 60)
```

`and` 和 `or` 有短路特性。

**短路** 的意思是：如果只看第一个条件就已经能决定结果，Python 就不会继续计算后面的条件。

例如：

```python
x = 0
if x != 0 and 10 / x > 1:
    print("ok")
```

因为 `x != 0` 是 `False`，整个 `and` 表达式一定是 `False`，所以 Python 不会继续计算 `10 / x`，也就避免了除零错误。

### 相等运算符

Python 中要区分 `is` 和 `==`。

```python
a is b
a == b
```

区别：

- **`is` 判断两个名字是否引用同一个对象**
- **`==` 判断两个对象的值是否相等**

示例：

```python
a = [1, 2, 3]
b = [1, 2, 3]

print(a == b)  # True
print(a is b)  # False
```

`a` 和 `b` 的内容相同，所以 `a == b` 是 `True`。但它们是两个不同列表对象，所以 `a is b` 是 `False`。

如果这样写：

```python
a = [1, 2, 3]
b = a

print(a == b)  # True
print(a is b)  # True
```

这时 `a` 和 `b` 引用同一个对象，所以 `a is b` 也是 `True`。

日常写代码时，大多数判断值是否相等用 `==`。只有当你明确要判断“是不是同一个对象”时，才用 `is`。

### 比较运算符

常见比较运算符：

```python
<    # 小于
<=   # 小于等于
>    # 大于
>=   # 大于等于
```

示例：


### 算术运算符

常见算术运算符：

```python
+    # 加
-    # 减
*    # 乘
/    # 除
//   # 整除
%    # 取余
**   # 幂
```

示例：

```python

print(7 // 3)   # 2
print(7 % 3)    # 1
print(2 ** 3)   # 8
```

其中 `%` 在算法中很常见。它可以用来：

- 判断奇偶
- 实现循环下标
- 做哈希表的压缩函数
- 处理周期性问题

例如：


如果 `k` 除以 2 的余数是 0，说明 `k` 是偶数。

### 运算符优先级

复合表达式中，Python 会按优先级决定先算什么。

例如：

```python
result = 2 + 3 * 4
```

结果是 `14`，因为乘法比加法优先级高。

如果想改变顺序，使用括号：

```python
result = (2 + 3) * 4
```

结果是 `20`。

学习算法时，不建议过度依赖记忆复杂优先级。更好的习惯是：**当表达式稍微复杂时，主动加括号，让代码含义更清楚**。

### 2. 变量名不是盒子，而是标签

很多初学者会把变量理解成“盒子”，但在 Python 中，更准确的理解是：

变量名是贴在对象上的标签。

例如：

```python
x = 10
y = x
```

这里发生的事情是：

- 创建或取得一个整数对象 `10`
- 名字 `x` 指向这个对象
- 名字 `y` 也指向同一个对象

再看列表：

```python
a = [1, 2, 3]
b = a
b.append(4)

print(a)
```

输出是：

```python
[1, 2, 3, 4]
```

原因是 `a` 和 `b` 指向同一个列表对象。`b.append(4)` 修改了这个列表本身，所以通过 `a` 也能看到变化。

这件事非常重要。后面学习链表、树、图时，我们会频繁处理“引用”“节点”“指针式关系”。Python 里的对象引用就是理解这些结构的入口。

## 复杂度分析

今天重点掌握两个结论：

```python
def is_even(k):
    return k % 2 == 0
```

这个函数不管输入多大，都只做一次取余和一次比较，所以时间复杂度是 `O(1)`。

```python
def find_max(data):
    biggest = data[0]
    for value in data:
        if value > biggest:
            biggest = value
    return biggest
```

这个函数需要遍历整个列表。列表有 `n` 个元素，就要检查 `n` 个值，所以时间复杂度是 `O(n)`。

空间复杂度方面，它只额外用了 `biggest` 和 `value` 这几个变量，没有随着列表变长创建更多辅助空间，所以是 `O(1)`。

## 时间复杂度是什么

**时间复杂度**用来描述：
当输入规模 `n` 变大时，算法运行时间增长得有多快。

它不关心程序具体运行了几秒，而是关心**运行次数和数据规模之间的关系**。

------

### 例子 1：O(1)

```python
arr = [10, 20, 30, 40]
print(arr[0])
```

无论数组有 4 个元素，还是 400 万个元素，都只取第一个元素。

所以时间复杂度是：

```text
O(1)
```

意思是：**常数时间复杂度**。

------

### 例子 2：O(n)

```python
arr = [10, 20, 30, 40]

for x in arr:
    print(x)
```

如果数组有 `n` 个元素，就循环 `n` 次。

所以时间复杂度是：

```text
O(n)
```

意思是：**线性时间复杂度**。

------

### 例子 3：O(n²)

```python
arr = [10, 20, 30, 40]

for i in arr:
    for j in arr:
        print(i, j)
```

外层循环 `n` 次，内层循环每次也 `n` 次。

总次数：

```text
n × n = n²
```

所以时间复杂度是：

```text
O(n²)
```

意思是：**平方时间复杂度**。

------

### 常见时间复杂度从快到慢

| 时间复杂度   | 名称           | 例子               |
| ------------ | -------------- | ------------------ |
| `O(1)`       | 常数复杂度     | 直接访问数组元素   |
| `O(log n)`   | 对数复杂度     | 二分查找           |
| `O(n)`       | 线性复杂度     | 遍历数组           |
| `O(n log n)` | 线性对数复杂度 | 快速排序、归并排序 |
| `O(n²)`      | 平方复杂度     | 双重循环           |
| `O(2ⁿ)`      | 指数复杂度     | 暴力枚举子集       |

------

### 核心判断方法

看代码中**随着 n 增长，主要操作执行了多少次**。

例如：

```python
for i in range(n):
    print(i)
```

执行 `n` 次：

```text
O(n)
for i in range(n):
    for j in range(n):
        print(i, j)
```

执行 `n²` 次：

```text
O(n²)
```

------

### 一句话理解

**时间复杂度就是用数学形式描述算法“随着数据量变大，会慢得多快”。**



## 空间复杂度是什么

**空间复杂度**表示：算法运行时，随着输入规模 `n` 变大，额外占用的内存空间增长得有多快。

它关注的是：

> 这个算法除了输入数据本身之外，还需要多少额外空间。

常用写法：

```text
O(1)
O(n)
O(n²)
O(log n)
```

------

### 1. O(1)：常数空间

无论 `n` 多大，只使用固定数量的变量。

```python
def find_max(nums):
    max_num = nums[0]

    for num in nums:
        if num > max_num:
            max_num = num

    return max_num
```

这里额外用了：

```python
max_num
num
```

变量数量不会随着 `nums` 变大而增加。

所以空间复杂度是：

```text
O(1)
```

------

### 2. O(n)：线性空间

随着输入规模 `n` 变大，额外空间也按比例变大。

```python
def copy_list(nums):
    result = []

    for num in nums:
        result.append(num)

    return result
```

如果 `nums` 有 `n` 个元素，`result` 也会存 `n` 个元素。

所以空间复杂度是：

```text
O(n)
```

------

### 3. O(n²)：平方空间

常见于二维数组、矩阵。

```python
def create_matrix(n):
    matrix = []

    for i in range(n):
        row = []
        for j in range(n):
            row.append(0)
        matrix.append(row)

    return matrix
```

当 `n = 3` 时，矩阵有：

```text
3 × 3 = 9 个元素
```

当 `n = 100` 时，矩阵有：

```text
100 × 100 = 10000 个元素
```

所以空间复杂度是：

```text
O(n²)
```

------

### 4. 递归也会占空间

递归函数会占用调用栈空间。

```python
def factorial(n):
    if n == 1:
        return 1
    return n * factorial(n - 1)
```

当执行：

```python
factorial(5)
```

调用过程是：

```text
factorial(5)
factorial(4)
factorial(3)
factorial(2)
factorial(1)
```

递归深度是 `n`，所以空间复杂度是：

```text
O(n)
```

------

### 空间复杂度和时间复杂度的区别

| 类型       | 关注点       | 问题             |
| ---------- | ------------ | ---------------- |
| 时间复杂度 | 运行时间增长 | 程序跑得快不快   |
| 空间复杂度 | 内存占用增长 | 程序占内存多不多 |

例如：

```python
def sum_nums(nums):
    total = 0

    for num in nums:
        total += num

    return total
```

时间复杂度：

```text
O(n)
```

因为要遍历所有元素。

空间复杂度：

```text
O(1)
```

因为只用了 `total` 和 `num` 这几个变量。

------

### 判断空间复杂度的步骤

1. 看有没有新建数组、列表、字典、集合。
2. 看新建的数据结构大小是否和 `n` 有关。
3. 看有没有递归。
4. 忽略固定数量的普通变量。

------

### 总结

```text
空间复杂度 = 算法运行时额外占用的内存空间随 n 增长的趋势
```

常见情况：

| 代码特征                | 空间复杂度 |
| ----------------------- | ---------- |
| 只用几个变量            | O(1)       |
| 新建一个长度为 n 的列表 | O(n)       |
| 新建 n × n 的二维数组   | O(n²)      |
| 递归 n 层               | O(n)       |
| 二分查找递归 log n 层   | O(log n)   |
